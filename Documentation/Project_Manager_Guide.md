## Introduction

This guide is designed for Project Managers using Project Nidus to plan, execute, and deliver successful projects.

**Key Capabilities:**
- Create and manage projects across multiple methodologies
- Plan and schedule tasks with dependencies
- Allocate and manage resources
- Track progress and performance
- Manage risks and issues
- Generate comprehensive reports

---

## Getting Started

### Your First Project

1. Navigate to **Projects** from the main menu
2. Click **Create New Project**
3. Enter project details:
   - Project name
   - Description
   - Start date
   - Methodology (Structured PM, Scrum, or Kanban)
4. Click **Create**

### Project Dashboard

Each project has a dashboard showing:
- Project overview and status
- Recent activity
- Upcoming tasks
- Key metrics
- Team members

---

## Creating Projects

### Project Details

When creating a project, provide:
- **Name**: Clear, descriptive project name
- **Description**: Project scope and objectives
- **Start Date**: Project kickoff date
- **End Date**: Target completion date (optional)
- **Methodology**: Choose Structured PM, Scrum, or Kanban
- **Project Manager**: Assign yourself or another PM
- **Stakeholders**: Add key stakeholders

### Project Templates

Use project templates for common project types to speed up setup.

## Methodology Selection

### Structured PM (PRINCE2)

Best for:
- Large, complex projects
- Projects requiring stage gates and approvals
- Projects with formal documentation requirements

Key features:
- 7 processes (Starting Up, Initiating, Directing, Controlling, Managing Product Delivery, Managing Stage Boundary, Closing)
- Stage gates and approvals
- Comprehensive documentation

### Scrum

Best for:
- Agile software development
- Projects with changing requirements
- Iterative delivery

Key features:
- Sprints and sprint planning
- Product backlog
- Sprint board
- Burndown charts

### Kanban

Best for:
- Continuous delivery
- Process improvement
- Visual workflow management

Key features:
- Kanban board
- WIP limits
- Flow metrics
- Cumulative flow diagrams

## Task Management

### Creating Tasks

1. Navigate to your project
2. Go to **Tasks**
3. Click **Create Task**
4. Enter task details:
   - Task name
   - Description
   - Assignee
   - Due date
   - Estimated effort
   - Dependencies (if any)

### Task Dependencies

Create task dependencies to:
- Ensure proper task sequencing
- Automatically calculate project timeline
- Identify critical path

### Gantt Chart

Use the Gantt chart to:
- Visualize project timeline
- Adjust task dates
- View dependencies
- Track progress

---

## Resource Allocation

### Assigning Resources

1. Navigate to **Resources** in your project
2. View resource availability
3. Assign team members to tasks
4. Check for resource conflicts

### Resource Capacity

Monitor resource capacity to:
- Avoid overallocation
- Balance workloads
- Plan for future needs

---

## Timeline Planning

### Creating Project Schedule

1. Define all tasks
2. Set task dependencies
3. Assign resources
4. Review Gantt chart
5. Adjust dates as needed
6. Set baseline for tracking

### Critical Path

The critical path shows tasks that, if delayed, will delay the entire project. Focus on managing these tasks carefully.

## Risk and Issue Management

### Risk Management

1. Navigate to **Risks** in your project
2. Identify and document risks
3. Assess probability and impact
4. Define mitigation strategies
5. Assign risk owners
6. Monitor and update risks regularly

### Issue Management

1. Navigate to **Issues** in your project
2. Document issues as they arise
3. Assign owners
4. Track resolution
5. Escalate critical issues

### RAID Log

The RAID Log (Risks, Assumptions, Issues, Dependencies) provides a consolidated view of project concerns.

## Best Practices

### Project Planning

- Start with clear objectives
- Break down work into manageable tasks
- Identify dependencies early
- Involve stakeholders from the start
- Set realistic timelines

### Execution

- Hold regular status meetings
- Update progress frequently
- Communicate proactively
- Address risks and issues promptly
- Adapt as needed

### Monitoring

- Review dashboards regularly
- Track key metrics
- Compare actual vs. planned
- Adjust plans when necessary
