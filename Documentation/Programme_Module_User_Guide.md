# Programme Module — User Guide

## Overview

The **Programme** module lets you coordinate multiple related projects under a single programme, monitor progress and benefits, and analyse inter‑project dependencies.

This guide covers the **Platform** routes under `/platform/programme/*` and explains how each page and tab works from an end‑user perspective.

---

## Accessing the Programme Module

- **Sidebar section:** `Programme`
- **Platform base route:** `/platform/programme`

From the Programme section you can access:
- **All Programmes**
- **Programme Dashboard**
- **Programme Projects**
- **Dependencies**
- **Benefits**
- **Timeline**
- **Reports**

All pages are theme‑aware (dark by default), mobile‑responsive, and require a valid Platform login with appropriate permissions.

---

## All Programmes — `/platform/programme`

**Purpose:** High‑level list of all programmes in your organisation.

**Key features:**
- **Programme cards**:
  - Name, code, type, status, and brief description.
  - Status colouring (e.g. active, planning, on‑hold).
- **Search bar**:
  - Filters by programme name, code, or description.
- **Quick stats row**:
  - Total programmes.
  - Count of active / planning / on‑hold programmes.
- **Create Programme button**:
  - Opens the programme create flow (currently a thin wrapper around the existing form).
- **Navigation:**
  - Clicking a card opens the **single programme view** at `/platform/programme/:id`.

Use this page to locate a programme quickly and drill down into its details.

---

## Single Programme View — `/platform/programme/:id`

**Purpose:** Complete view of a single programme, with dashboard and detailed tabs.

### Header

The header shows:
- Programme name and code.
- Short description.
- Export buttons:
  - PowerPoint, Word, Excel, CSV, XML, JSON, and Print.
- **Edit** button:
  - Opens the programme form in a modal so you can update key fields without leaving the page.

### Tabs

The single programme page provides six tabs.

#### 1. Dashboard

**Component:** `ProgrammeDashboard`

Shows an at‑a‑glance summary:
- Total, active, completed projects.
- Programme progress and health score.
- Benefits realization percentage.
- Milestone and dependency indicators (including critical/at‑risk items).
- A quick list of top projects by priority and status.

Use this tab for an immediate view of overall programme health and progress.

#### 2. Projects

**Data source:** `getProgrammeProjects(programmeId)`

Displays a table of all projects currently assigned to the programme:
- Project name and code.
- Project status.
- Programme priority (e.g. critical/high/normal).

Use this tab to check which projects are in the programme and how they are performing.

#### 3. Dependencies

**Data source:** `getProgrammeDependencies(programmeId)`  
**Component:** `DependencyMapVisualization`

Shows:
- Summary cards:
  - Total dependencies.
  - Critical dependencies.
  - At‑risk / blocked dependencies.
- Visual list of dependency links:
  - Source project → target project.
  - Dependency type and status.
  - Critical / at‑risk badges.

Use this tab to understand cross‑project dependencies and identify risk hot‑spots.

#### 4. Benefits

**Data source:** `getProgrammeBenefits(programmeId)`  
**Component:** `BenefitsRealizationChart`

Provides:
- A progress bar of **benefits realized vs total**.
- Breakdown cards:
  - Realized, in progress, and not started counts.
- **Value realization**:
  - Actual vs expected benefit value, shown as a percentage.

Use this tab to track whether the programme is on track to deliver its promised benefits.

#### 5. Timeline

**Data sources:**
- `getProgrammeMilestones(programmeId)`
- `getProgrammeProjects(programmeId)`  
**Component:** `ProgrammeTimelineView`

Visualises the programme’s time dimension:
- Overall programme date range.
- Milestones plotted on a horizontal timeline, coloured by status:
  - Completed, overdue, upcoming.
- High‑level statistics:
  - Total, completed, and overdue milestones.

Use this tab to see whether major milestones are on track and how they align with project schedules.

#### 6. Reports

**Data source:** `getProgrammeReports(programmeId)`

Lists formal programme‑level reports:
- Report date.
- Report type (e.g. highlight, exception, end stage, end project).
- Status (draft, in review, approved, archived).
- Generated by / approved by.

Use this tab as a central register for all structured reports associated with the programme.

## Global Programme Pages

The Programme submenu provides several global views that work across all programmes.

### 1. Programme Dashboard — `/platform/programme/dashboard`

**Component:** `ProgrammeDashboardOverview`

Shows a tiled view of all programmes (per account), using roll‑up data:
- Per‑programme cards:
  - Name and code.
  - Total and active projects.
  - Progress percentage with a bar.
  - Benefits realization percentage.
  - Overall health score.

Use this page to compare programmes side‑by‑side and identify which ones need attention.

### 2. Programme Projects — `/platform/programme/projects`

**Component:** `ProgrammeProjectsPage`

Provides:
- Programme dropdown (using `getProgrammeList()`).
- When a programme is selected:
  - Table of all assigned projects (name, code, status, programme priority).

Use this page to see the project portfolio for a single programme without leaving a global context.

### 3. Programme Dependencies — `/platform/programme/dependencies`

**Component:** `ProgrammeDependenciesPage`

Provides:
- Programme dropdown.
- For the selected programme:
  - `DependencyMapVisualization` to show dependency network and risk levels.

Use this page for focused dependency analysis per programme.

### 4. Programme Benefits — `/platform/programme/benefits`

**Component:** `ProgrammeBenefitsPage`

Provides:
- Programme dropdown.
- `BenefitsRealizationChart` for that programme.
- Table of benefits (name, category, type, status).

Use this page to compare benefit profiles across programmes and inspect a specific programme’s benefits in more detail.

### 5. Programme Timeline — `/platform/programme/timeline`

**Component:** `ProgrammeTimelinePage`

Provides:
- Programme dropdown.
- `ProgrammeTimelineView` fed with:
  - programme details.
  - milestones.
  - related projects.

Use this to focus only on scheduling and milestone alignment for a single programme.

### 6. Programme Reports — `/platform/programme/reports`

**Component:** `ProgrammeReportsPage`

Provides:
- Programme dropdown.
- Filters:
  - Report type (highlight, exception, end stage, end project).
  - Status (draft, in review, approved, archived).
- Table of reports matching the filters.

Use this page as a global view of all structured reports produced for a given programme.

## Simulator Parity (High‑Level)

The Simulator module has a parallel **Practice Programme** feature using the `sim` schema:
- **List:** `/simulator/practice-programme`
- **Detail:** `/simulator/practice-programme/:programmeId`

Simulator practice programmes are stored separately from Platform data and are safe for experimentation. In future iterations, Simulator dashboards (practice dependencies, benefits, timeline, reports) can mirror the Platform views using sim‑specific tables and the `simDb` client.
