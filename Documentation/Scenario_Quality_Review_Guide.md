# Scenario Quality Review Guide

## Review Process

### Initial Review
1. **Content Accuracy**
   - [ ] Scenario reflects real-world project management situations
   - [ ] Technical details are accurate
   - [ ] Industry-specific information is correct
   - [ ] Methodology application is appropriate

2. **Learning Objectives**
   - [ ] Learning objectives are clear and measurable
   - [ ] Objectives align with scenario difficulty
   - [ ] Objectives are achievable within scenario duration

3. **Difficulty Calibration**
   - [ ] Difficulty matches target audience
   - [ ] Challenge level is appropriate for skill level
   - [ ] Progression is logical

### Event Quality

4. **Event Realism**
   - [ ] Events are realistic and plausible
   - [ ] Events reflect common project management challenges
   - [ ] Timing of events is appropriate
   - [ ] Events are relevant to scenario context

5. **Decision Options**
   - [ ] Multiple viable options are provided
   - [ ] Options are distinct and meaningful
   - [ ] Best option is not always obvious
   - [ ] Options reflect real decision-making scenarios

6. **Feedback Quality**
   - [ ] Feedback explains why decisions were good/bad
   - [ ] Feedback provides learning insights
   - [ ] Feedback is constructive and educational
   - [ ] Feedback references best practices

### Technical Quality

7. **Structure & Flow**
   - [ ] Scenario phases are well-defined
   - [ ] Transitions between phases are smooth
   - [ ] Event sequence is logical
   - [ ] Scenario has clear beginning, middle, and end

8. **Completeness**
   - [ ] All required fields are populated
   - [ ] All phases have events
   - [ ] All events have decisions
   - [ ] All decisions have feedback

9. **Consistency**
   - [ ] Terminology is consistent
   - [ ] Character names/roles are consistent throughout
   - [ ] Project details are consistent
   - [ ] Methodology application is consistent

### User Experience

10. **Clarity**
    - [ ] Instructions are clear
    - [ ] Scenario description is understandable
    - [ ] Events are clearly described
    - [ ] Decisions are clearly worded

11. **Engagement**
    - [ ] Scenario is engaging and interesting
    - [ ] Story is compelling
    - [ ] User feels invested in outcome
    - [ ] Scenario maintains user interest

12. **Accessibility**
    - [ ] Content is accessible to all users
    - [ ] Language is clear and professional
    - [ ] No offensive or inappropriate content
    - [ ] Cultural sensitivity is considered

## Review Checklist by Scenario Type

### Beginner Scenarios
- [ ] Concepts are clearly explained
- [ ] Terminology is defined
- [ ] Guidance is provided
- [ ] Learning curve is gentle
- [ ] Success is achievable

### Intermediate Scenarios
- [ ] Builds on beginner knowledge
- [ ] Introduces complexity gradually
- [ ] Requires strategic thinking
- [ ] Multiple valid approaches exist
- [ ] Challenges are meaningful

### Expert Scenarios
- [ ] Requires advanced knowledge
- [ ] Complex decision-making required
- [ ] Multiple competing priorities
- [ ] Real-world complexity reflected
- [ ] High-level strategic thinking needed

## Quality Scoring

### Scoring Criteria (1-5 scale)

**Content Quality (25%)**
- Accuracy: 1-5
- Realism: 1-5
- Relevance: 1-5

**Learning Value (25%)**
- Learning objectives: 1-5
- Educational value: 1-5
- Skill development: 1-5

**Technical Quality (25%)**
- Structure: 1-5
- Completeness: 1-5
- Consistency: 1-5

**User Experience (25%)**
- Clarity: 1-5
- Engagement: 1-5
- Accessibility: 1-5

### Minimum Scores
- **Beginner**: 3.5/5 overall
- **Intermediate**: 4.0/5 overall
- **Expert**: 4.5/5 overall

## Review Workflow

1. **Assign Reviewer**
   - Match reviewer expertise to scenario type
   - Ensure reviewer has time for thorough review

2. **Initial Review**
   - Complete review checklist
   - Document issues and concerns
   - Score scenario quality

3. **Feedback to Creator**
   - Provide constructive feedback
   - Highlight strengths
   - Suggest improvements

4. **Revision Review**
   - Review revised scenario
   - Verify issues are addressed
   - Re-score if significant changes

5. **Approval**
   - Final quality check
   - Approve for publication
   - Update scenario status

## Common Issues

### Content Issues
- Unrealistic events or situations
- Inaccurate technical information
- Poor alignment with learning objectives
- Inappropriate difficulty level

### Technical Issues
- Missing or incomplete fields
- Inconsistent information
- Broken event sequences
- Missing feedback

### UX Issues
- Unclear instructions
- Confusing decision options
- Poor feedback quality
- Low engagement
