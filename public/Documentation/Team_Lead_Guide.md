## Introduction

This guide is designed for Team Leads managing teams within Project Nidus projects.

**Key Responsibilities:**
- Manage team members and assignments
- Plan and execute sprints (Scrum)
- Manage Kanban boards
- Track team progress
- Allocate resources effectively
- Foster team collaboration

---

## Getting Started

### Team Overview

As a Team Lead, you can:
- View all projects your team is assigned to
- Access team-specific dashboards
- Manage task assignments
- Track team performance

### Team Dashboard

The team dashboard shows:
- Team members and their current assignments
- Upcoming tasks
- Team workload
- Progress metrics
- Recent activity

---

## Team Management

### Viewing Team Members

1. Navigate to **Teams** or your project
2. View team members list
3. See each member's:
   - Current assignments
   - Workload
   - Availability
   - Skills and expertise

### Team Capacity

Monitor team capacity to:
- Balance workloads
- Identify overloaded team members
- Plan for capacity increases
- Allocate work fairly

---

## Task Assignment

### Assigning Tasks

1. Navigate to project **Tasks**
2. Select a task
3. Click **Assign**
4. Choose team member(s)
5. Set due date and priority
6. Add instructions or notes

### Bulk Assignment

Assign multiple tasks at once:
1. Select multiple tasks
2. Click **Bulk Assign**
3. Choose team members
4. Set common due date
5. Confirm assignment

---

## Sprint Planning (Scrum)

### Creating Sprints

1. Navigate to project **Scrum > Sprint Planning**
2. Click **New Sprint**
3. Define sprint details:
   - Sprint name
   - Start date
   - End date
   - Sprint goal
4. Select tasks from product backlog
5. Assign team members
6. Start sprint

### Sprint Board

During sprint execution:
- View tasks on sprint board
- Update task status
- Track sprint progress
- Monitor burndown chart

### Daily Scrum

Facilitate daily stand-ups:
1. Navigate to **Scrum > Daily Scrum**
2. Review yesterday's progress
3. Plan today's work
4. Identify blockers
5. Update task status

---

## Kanban Board Management

### Setting Up Kanban

1. Navigate to project **Kanban**
2. Configure board columns
3. Set WIP (Work In Progress) limits
4. Define workflow stages

### Managing Workflow

- Move tasks between columns
- Respect WIP limits
- Identify bottlenecks
- Optimize flow

### Flow Metrics

Monitor flow metrics:
- Cycle time
- Lead time
- Throughput
- Cumulative flow diagram

### Team Status Reports

### Velocity Tracking (Scrum)

Track team velocity:
