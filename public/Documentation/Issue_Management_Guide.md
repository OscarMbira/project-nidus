# Issue Management User Guide

## Overview

Issue Management is a universal module for logging, tracking, and resolving issues, impediments, and blockers across all project methodologies. This guide covers creating issues, tracking status, linking to work items, and resolving issues.

## Accessing Issue Management

1. Navigate to your project from the **Projects** page
2. Click on **Issue Management** in the project navigation menu
3. View issues list and dashboard

## Understanding Issues

### What is an Issue?

An issue is any event, problem, or concern that:
- Affects project progress
- Requires attention or resolution
- May impact project objectives
- Needs tracking and management

### Issue Types

- **Bug**: Software defect or error
- **Enhancement**: Feature improvement request
- **Task**: Work item or action
- **Question**: Inquiry requiring answer
- **Blocker**: Item preventing progress
- **Risk**: Potential problem
- **Other**: Miscellaneous issue

## Creating Issues

### Creating a New Issue

1. Click **+ Create Issue** button
2. Fill in issue details:
   - **Issue Title** (required): Brief description
   - **Issue Code**: Unique identifier (auto-generated if blank)
   - **Description** (required): Detailed description
   - **Issue Type**: Bug, Enhancement, Task, etc.
   - **Priority**: Low, Medium, High, Critical
   - **Severity**: Low, Medium, High, Critical
   - **Assigned To**: Team member responsible
   - **Due Date**: Target resolution date
   - **Category**: Issue category
3. Click **Save**

### Issue Details

**Basic Information**:
- Title and code
- Type and category
- Description
- Status

**Priority and Severity**:
- Priority: Urgency of resolution
- Severity: Impact if not resolved
- Combined assessment

**Assignment**:
- Assigned team member
- Reporter
- Due date

**Impact**:
- Impact description
- Affected areas
- Consequences

**Links**:
- Linked tasks
- Linked work packages
- Linked user stories
- Linked Kanban cards

## Issue Status Workflow

### Status Types

1. **New**: Issue just created
2. **Assigned**: Issue assigned to team member
3. **In Progress**: Work on issue has started
4. **Resolved**: Issue resolved, awaiting verification
5. **Closed**: Issue verified and closed
6. **Reopened**: Issue reopened after closure

### Updating Status

1. Click on issue
2. Click **Update Status**
3. Select new status
4. Add status change comments
5. Save changes

### Status Transitions

**New → Assigned**:
- Issue assigned to team member
- Assignment date recorded

**Assigned → In Progress**:
- Work on issue begins
- Progress updates available

**In Progress → Resolved**:
- Issue resolution complete
- Resolution details recorded

**Resolved → Closed**:
- Issue verified as resolved
- Closure confirmed

**Closed → Reopened**:
- Issue not fully resolved
- Requires additional work

## Linking Issues

### Linking to Work Items

Issues can be linked to:
- **Tasks**: Related project tasks
- **Work Packages**: Structured PM work packages
- **User Stories**: Scrum user stories
- **Kanban Cards**: Kanban board cards

### Adding Links

1. Open issue details
2. Go to **Links** section
3. Click **Link to Task/Work Package/User Story/Card**
4. Select item from dropdown
5. Save link

### Viewing Linked Items

1. View issue details
2. See linked items in **Links** section
3. Click on linked item to view details
4. Navigate to linked item's page

## Issue Filtering and Search

### Filtering Issues

1. Click **Filter** button
2. Select filter criteria:
   - **Status**: Filter by issue status
   - **Priority**: Filter by priority level
   - **Type**: Filter by issue type
   - **Assigned To**: Filter by assignee
   - **Date Range**: Filter by creation or due date
3. Click **Apply**

### Searching Issues

1. Use **Search** box
2. Search by:
   - Issue title
   - Issue code
   - Description
   - Comments
3. Results update as you type

### Saved Filters

1. Configure filters
2. Click **Save Filter**
3. Name the filter
4. Access from **Saved Filters** dropdown

## Issue Comments

### Adding Comments

1. Open issue details
2. Go to **Comments** section
3. Enter comment text
4. Click **Add Comment**
5. Comment appears in timeline

### Comment Features

- **Timeline View**: Comments in chronological order
- **User Attribution**: Shows comment author
- **Timestamps**: Date and time of comments
- **Rich Text**: Formatting support

## Issue Attachments

### Adding Attachments

1. Open issue details
2. Go to **Attachments** section
3. Click **+ Add Attachment**
4. Select file(s)
5. Upload files
6. Files attached to issue

### Attachment Types

- Documents (PDF, Word, Excel)
- Images (PNG, JPG)
- Screenshots
- Log files
- Other relevant files

## Issue Resolution

### Resolving Issues

1. Work on issue to resolution
2. Update issue status to **Resolved**
3. Add resolution details:
   - Resolution description
   - Steps taken
   - Root cause (if applicable)
   - Prevention measures
4. Save resolution

### Closing Issues

1. Verify issue is resolved
2. Update status to **Closed**
3. Add closure comments
4. Confirm closure
5. Issue moved to closed list

### Reopening Issues

1. If issue not fully resolved
2. Update status to **Reopened**
3. Add reopening reason
4. Reassign if needed
5. Continue resolution process

## Issue Dashboard

### Dashboard Metrics

View issue dashboard:
- **Total Issues**: All issues in project
- **Open Issues**: Issues not closed
- **By Status**: Issues by status
- **By Priority**: Issues by priority
- **By Type**: Issues by type
- **Overdue**: Issues past due date

### Charts and Reports

- **Status Distribution**: Pie chart of statuses
- **Priority Distribution**: Bar chart of priorities
- **Trend Analysis**: Issues over time
- **Resolution Time**: Average time to resolve

## Best Practices

### Creating Issues

1. **Be Specific**: Clear and specific titles
2. **Provide Details**: Comprehensive descriptions
3. **Set Priority**: Appropriate priority level
4. **Assign Quickly**: Assign to team member promptly
5. **Set Due Dates**: Realistic due dates

### Managing Issues

1. **Update Regularly**: Keep status current
2. **Add Comments**: Document progress
3. **Link Related Items**: Link to related work
4. **Track Resolution**: Monitor resolution progress
5. **Close Promptly**: Close when resolved

### Issue Resolution

1. **Investigate Thoroughly**: Understand root cause
2. **Document Solution**: Record resolution steps
3. **Verify Resolution**: Ensure issue truly resolved
4. **Learn from Issues**: Identify prevention measures
5. **Share Knowledge**: Document for team

## Troubleshooting

### Common Issues

**Issue Not Appearing**:
- Check filters applied
- Verify project selection
- Refresh page
- Check permissions

**Cannot Assign Issue**:
- Verify team member is project member
- Check permissions
- Contact administrator

**Status Not Updating**:
- Refresh page
- Check if status transition valid
- Verify permissions
- Contact administrator
