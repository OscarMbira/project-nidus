# Structured PM - Controlling a Stage (CS) User Guide

## Overview

Controlling a Stage (CS) is a core Structured PM process that manages the day-to-day work within a project stage. This guide covers work package management, checkpoint reports, highlight reports, and stage tolerance monitoring.

## Accessing Controlling a Stage

1. Navigate to your project from the **Projects** page
2. Click on **Structured PM** in the project navigation menu
3. Select **Controlling a Stage**
4. Choose the active stage

## Understanding Stages

### Stage Concept

- **Stage**: A distinct period in the project lifecycle
- **Stage Manager**: Person responsible for stage delivery
- **Stage Plan**: Detailed plan for the stage
- **Stage Tolerances**: Acceptable deviations from plan

### Stage Lifecycle

1. **Authorize Stage**: Stage approved to begin
2. **Execute Stage**: Work packages executed
3. **Monitor Stage**: Progress tracked and reported
4. **Complete Stage**: Stage finished and reviewed

## Work Packages

### What are Work Packages?

Work Packages define the work to be done during a stage. They are:
- Assigned to team members
- Have clear deliverables
- Include quality criteria
- Have defined acceptance criteria

### Creating Work Packages

1. Click **+ Create Work Package** button
2. Fill in details:
   - **Work Package Name** (required)
   - **Work Package Code**: Unique identifier
   - **Description**: Detailed description
   - **Assigned To**: Team member responsible
   - **Planned Start Date**
   - **Planned End Date**
   - **Estimated Effort**: Hours or days
   - **Deliverables**: What will be produced
3. Click **Save**

### Work Package Details

**Basic Information**:
- Name and code
- Description
- Status (Planned, Assigned, In Progress, Complete)
- Priority

**Assignment**:
- Assigned team member
- Assignment date
- Acceptance date

**Timeline**:
- Planned dates
- Actual dates
- Duration
- Progress percentage

**Deliverables**:
- List of deliverables
- Quality criteria
- Acceptance criteria
- Status of each deliverable

### Managing Work Packages

**Updating Status**:
1. Click on work package
2. Update status dropdown
3. Add progress notes
4. Save changes

**Tracking Progress**:
- Update percentage complete
- Log actual effort
- Update actual dates
- Add progress comments

**Completing Work Package**:
1. Ensure all deliverables complete
2. Verify quality criteria met
3. Update status to "Complete"
4. Record completion date

## Checkpoint Reports

### Purpose

Checkpoint Reports provide regular updates on stage progress to the Stage Manager. They are typically produced weekly or bi-weekly.

### Creating Checkpoint Reports

1. Click **+ Create Checkpoint Report**
2. Fill in report details:
   - **Report Date**: Date of report
   - **Reporting Period**: Start and end dates
   - **Work Package Status**: Status of each work package
   - **Progress Summary**: Overall progress
   - **Issues and Risks**: Current issues and risks
   - **Next Period Plan**: Planned work for next period
3. Click **Save**

### Report Contents

**Work Package Status**:
- List of all work packages
- Current status
- Progress percentage
- Planned vs. actual dates
- Issues or blockers

**Progress Summary**:
- Overall stage progress
- Completed work packages
- In-progress work packages
- Upcoming work packages

**Issues and Risks**:
- Current issues affecting work
- New risks identified
- Mitigation actions
- Escalation needs

**Next Period Plan**:
- Work packages to start
- Work packages to complete
- Key milestones
- Resource requirements

### Submitting Reports

1. Complete all required sections
2. Review for accuracy
3. Click **Submit Report**
4. Report sent to Stage Manager
5. Receive acknowledgment

## Highlight Reports

### Purpose

Highlight Reports provide summary information to the Project Board. They are typically produced monthly or at key stage milestones.

### Creating Highlight Reports

1. Click **+ Create Highlight Report**
2. Fill in report details:
   - **Report Date**: Date of report
   - **Reporting Period**: Start and end dates
   - **Stage Summary**: High-level stage status
   - **Key Achievements**: Major accomplishments
   - **Issues and Risks**: Summary of issues/risks
   - **Forecast**: Stage completion forecast
   - **Tolerance Status**: Tolerance deviations
3. Click **Save**

### Report Contents

**Stage Summary**:
- Overall stage status (Green, Amber, Red)
- Progress against plan
- Key metrics
- Stage health indicator

**Key Achievements**:
- Completed work packages
- Delivered products
- Milestones reached
- Quality achievements

**Issues and Risks Summary**:
- Number of open issues
- Number of open risks
- Critical items
- Escalation status

**Forecast**:
- Expected stage completion date
- Budget forecast
- Resource forecast
- Quality forecast

**Tolerance Status**:
- Time tolerance status
- Cost tolerance status
- Scope tolerance status
- Quality tolerance status

### Submitting to Project Board

1. Complete all sections
2. Review with Stage Manager
3. Click **Submit to Project Board**
4. Report available to board members
5. Board may request additional information

## Stage Tolerances

### Understanding Tolerances

Tolerances define acceptable deviations from the stage plan:
- **Time Tolerance**: Acceptable schedule variance
- **Cost Tolerance**: Acceptable budget variance
- **Scope Tolerance**: Acceptable scope changes
- **Quality Tolerance**: Acceptable quality variance

### Monitoring Tolerances

1. Go to **Tolerance Monitoring** section
2. View tolerance dashboard:
   - Current tolerance status
   - Variance from plan
   - Trend indicators
   - Warning alerts

### Tolerance Status Indicators

- **Green**: Within tolerance
- **Amber**: Approaching tolerance limit
- **Red**: Exceeded tolerance (escalation required)

### Escalating Tolerance Breaches

1. Identify tolerance breach
2. Document the variance
3. Create exception report
4. Escalate to Project Board
5. Request tolerance increase or stage replan

## Stage Completion

### Completing a Stage

1. **Verify Completion**:
   - All work packages complete
   - All deliverables accepted
   - Quality criteria met
   - Stage objectives achieved

2. **Stage Review**:
   - Review stage performance
   - Document lessons learned
   - Update project records
   - Prepare stage closure report

3. **Request Stage Closure**:
   - Submit closure request
   - Include completion evidence
   - Await Project Board approval
   - Close stage formally

### Stage Closure Report

**Contents**:
- Stage objectives achievement
- Work packages completed
- Deliverables produced
- Issues and risks resolved
- Lessons learned
- Recommendations for next stage

## Best Practices

### Work Package Management

1. **Clear Definition**: Define work packages clearly
2. **Realistic Estimates**: Base estimates on experience
3. **Regular Updates**: Update progress frequently
4. **Early Warning**: Flag issues early
5. **Quality Focus**: Don't compromise on quality

### Reporting

1. **Regular Reports**: Submit reports on schedule
2. **Accurate Data**: Ensure data is accurate
3. **Clear Communication**: Write clearly and concisely
4. **Action-Oriented**: Include actionable items
5. **Timely Submission**: Submit reports on time

### Tolerance Management

1. **Monitor Regularly**: Check tolerances frequently
2. **Early Warning**: Identify issues before breach
3. **Proactive Management**: Take action early
4. **Document Variance**: Record all variances
5. **Escalate Promptly**: Don't delay escalation

## Troubleshooting

### Common Issues

**Work Package Not Progressing**:
- Check if assigned correctly
- Verify resources available
- Identify blockers
- Escalate if needed

**Report Submission Issues**:
- Check all required fields
- Verify dates are valid
- Ensure work package data current
- Contact administrator

**Tolerance Breach**:
- Document the variance
- Identify root cause
- Create exception report
- Escalate to Project Board
