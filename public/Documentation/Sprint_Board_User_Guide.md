# Sprint Board User Guide

## Overview

The Sprint Board is a Scrum-specific Kanban board that visualizes work during a sprint. It helps teams track sprint progress, manage sprint backlog items, and ensure sprint goals are met. This guide covers sprint board usage, sprint management, and sprint ceremonies.

## Accessing the Sprint Board

1. Navigate to your project from the **Projects** page
2. Click on **Scrum** in the project navigation menu
3. Select **Sprint Board**
4. Choose an active sprint or create a new one

## Understanding Sprint Boards

### Board Structure

- **Sprint Backlog**: User stories and tasks in the current sprint
- **Columns**: Workflow stages (To Do, In Progress, Testing, Done)
- **Sprint Goal**: Displayed at the top of the board
- **Sprint Info**: Sprint dates, capacity, and progress

### Default Columns

- **Backlog**: Items not yet started
- **To Do**: Ready to start
- **In Progress**: Currently being worked on
- **Testing/QA**: Items in review or testing
- **Done**: Completed and accepted

## Sprint Management

### Creating a Sprint

1. Click **+ New Sprint** button
2. Fill in sprint details:
   - **Sprint Name**: e.g., "Sprint 1 - Q1 2025"
   - **Start Date**: Sprint start date
   - **End Date**: Sprint end date (typically 1-4 weeks)
   - **Sprint Goal**: High-level objective for the sprint
   - **Capacity**: Team capacity in story points or hours
3. Click **Create Sprint**

### Starting a Sprint

1. Select a sprint from the sprint list
2. Click **Start Sprint**
3. Confirm sprint details
4. Sprint becomes active

### Planning Sprint Backlog

1. Go to **Product Backlog**
2. Select user stories for the sprint
3. Click **Add to Sprint**
4. Or drag stories directly to sprint board
5. Break down stories into tasks if needed

### Sprint Backlog Items

**User Stories**:
- High-level features or requirements
- Written in user story format
- Have story points for estimation
- Can be broken into tasks

**Tasks**:
- Specific work items
- Assigned to team members
- Have time estimates
- Tracked on the board

## Working with Sprint Board

### Moving Items

**Drag and Drop**:
1. Click and hold a card
2. Drag to target column
3. Release to drop
4. Status updates automatically

**Quick Actions**:
1. Click on a card
2. Use status dropdown
3. Select new status
4. Card moves automatically

### Updating Progress

1. **Update Task Status**: Move cards between columns
2. **Update Story Points**: Click on story to update remaining points
3. **Add Comments**: Document progress or blockers
4. **Log Time**: Track actual time spent

### Sprint Burndown Chart

**Viewing Burndown**:
1. Click **Burndown Chart** tab
2. View sprint progress visualization
3. See ideal vs. actual burndown
4. Identify if sprint is on track

**Understanding Burndown**:
- **Ideal Line**: Expected progress (straight line)
- **Actual Line**: Real progress (may vary)
- **Remaining Work**: Story points or hours left
- **Days Remaining**: Time left in sprint

**Interpreting Burndown**:
- **Above Ideal**: Ahead of schedule
- **Below Ideal**: Behind schedule (action needed)
- **Steep Drop**: Large amount of work completed
- **Flat Line**: No progress (investigate blockers)

## Sprint Ceremonies

### Daily Scrum

**Accessing Daily Scrum**:
1. Click **Daily Scrum** in sprint menu
2. View today's standup
3. Answer three questions:
   - What did I do yesterday?
   - What will I do today?
   - Are there any blockers?

**Recording Standup**:
1. Select your name
2. Answer questions
3. Add blockers if any
4. Save standup notes
5. View team updates

**Timer Feature**:
- 15-minute timer available
- Helps keep standups focused
- Automatic time tracking

### Sprint Review

**Conducting Review**:
1. Click **Sprint Review** in sprint menu
2. Review completed work
3. Demo completed features
4. Gather stakeholder feedback

**Review Checklist**:
- [ ] All done items reviewed
- [ ] Demos prepared
- [ ] Stakeholders invited
- [ ] Feedback collected
- [ ] Product backlog updated

**Recording Feedback**:
1. Add feedback items
2. Categorize (positive, improvement, bug)
3. Link to product backlog items
4. Create action items

### Sprint Retrospective

**Accessing Retrospective**:
1. Click **Sprint Retrospective** in sprint menu
2. View retro board
3. Add items to categories:
   - **What Went Well**: Positive aspects
   - **What to Improve**: Areas for improvement
   - **Action Items**: Concrete next steps

**Participating in Retro**:
1. Add your thoughts to each category
2. Vote on important items
3. Discuss as a team
4. Create action items

**Action Items**:
- Convert improvement items to actions
- Assign owners
- Set due dates
- Track in next sprint

**Using Velocity**:

2. View comprehensive sprint summary:
   - Completed items

   - Velocity achieved
   - Team performance
   - Burndown analysis

**Report Contents**:
- Sprint goal achievement

## Best Practices

### Sprint Planning

1. **Set Clear Goals**: Define sprint goal upfront
2. **Right-Size Stories**: Ensure stories fit in sprint
3. **Estimate Accurately**: Use story points consistently
4. **Plan Capacity**: Don't overcommit
5. **Identify Dependencies**: Plan around dependencies

### During Sprint

1. **Daily Updates**: Keep board current
2. **Track Progress**: Update burndown regularly
3. **Address Blockers**: Resolve blockers quickly
4. **Maintain Focus**: Avoid scope creep
5. **Communicate**: Keep team informed

### Sprint Completion

1. **Review All Work**: Ensure everything is done
2. **Update Metrics**: Record velocity and metrics
3. **Conduct Retrospective**: Learn and improve
4. **Update Backlog**: Refine based on learnings
5. **Celebrate Success**: Acknowledge achievements

## Troubleshooting

### Common Issues

**Items not moving to Done**:
- Check if all tasks are complete
- Verify acceptance criteria met
- Ensure proper status update
- Check permissions

**Burndown not updating**:
- Verify story points are updated
- Check if items are marked complete
- Refresh burndown chart
- Contact administrator

**Sprint capacity exceeded**:
- Review committed items
- Consider moving items to next sprint
- Adjust sprint duration
- Re-estimate items

**Blockers not visible**:
- Ensure blockers are logged
- Check blocker visibility settings
- Verify permissions
- Refresh board

## Keyboard Shortcuts

- **S**: Start sprint
- **B**: View burndown
- **D**: Daily scrum
- **R**: Retrospective
- **V**: View velocity
- **Arrow Keys**: Navigate cards
- **Enter**: Open card
