# Executive Dashboard Metrics Recommendations

## Purpose

This document provides recommended high-level summary and alert metrics for a **Platform Executive Dashboard** covering:

1. Portfolio
2. Programmes
3. Projects

The goal is to move the dashboard from simply showing counts to becoming an **executive decision cockpit** that quickly answers:

- Where are we exposed?
- What needs urgent intervention?
- What is performing well?
- What is blocked or delayed?
- What decisions are pending?
- Where are governance or linkage gaps?

---

# 1. Portfolio Dashboard Metrics

Portfolio-level metrics should help executives answer:

> Are our strategic investments healthy, aligned, funded, and delivering value?

---

## 1.1 Portfolio Summary Metrics

| Metric | Purpose |
|---|---|
| Total Portfolios | Shows the number of portfolios currently created in the platform. |
| Active Portfolios | Shows portfolios currently in execution or active oversight. |
| Completed Portfolios | Shows closed portfolios and helps track maturity of delivery. |
| Portfolio Health Index | Overall health across schedule, budget, risk, scope, benefits, and governance. |
| Strategic Alignment Score | Measures how well portfolios align with business objectives and strategy. |
| Total Portfolio Value | Total approved investment value or budget across portfolios. |
| Budget Utilization % | Shows how much of the approved portfolio budget has been consumed. |
| Benefits Realization % | Tracks whether expected strategic benefits are being achieved. |
| Portfolio Risk Exposure | Aggregated risk level across programmes and projects within the portfolio. |
| Portfolio Dependency Count | Number of major dependencies across programmes and projects. |
| Governance Compliance % | Measures whether required governance documents, approvals, and reviews are complete. |
| Decision Items Pending | Number of executive decisions awaiting action. |
| Programmes Linked to Portfolio | Number of programmes correctly linked to portfolios. |
| Programmes Without Portfolio | Number of programmes requiring portfolio alignment. |
| Portfolio Coverage % | Percentage of programmes that are linked to a portfolio. |

---

## 1.2 Portfolio Alert Metrics

| Alert | Why It Matters |
|---|---|
| Portfolio Over Budget | Warns executives where investment control is failing. |
| Benefits Not Tracking | Indicates that the portfolio may not deliver expected business value. |
| Strategic Misalignment | Flags portfolios not clearly linked to business objectives. |
| High-Risk Portfolio | Highlights portfolios with too many critical risks or failing programmes. |
| Stalled Portfolio | No recent movement, approvals, or progress within a defined period. |
| Unlinked Programmes | Programmes not properly assigned to a portfolio. |
| Delayed Executive Decisions | Decisions pending beyond agreed SLA. |
| Governance Gaps | Missing business case, mandate, approvals, reports, or reviews. |
| Funding Exception | Portfolio funding is insufficient, delayed, or not approved. |
| Dependency Hotspot | Too many unresolved cross-programme or cross-project dependencies. |

# 2. Programme Dashboard Metrics

Programme-level metrics should help executives answer:

> Are related projects coordinated properly, delivering outcomes, and managing dependencies?

---

## 2.1 Programme Summary Metrics

| Metric | Purpose |
|---|---|
| Total Programmes | Shows total number of programmes in the platform. |
| Active Programmes | Shows programmes currently in execution. |
| Planning Programmes | Shows programmes still being defined or prepared. |
| Completed Programmes | Shows programmes formally closed. |
| Programme Health Index | Aggregated health of all projects under the programme. |
| Programme Delivery Progress % | Overall delivery progress across linked projects. |
| Project Completion Rate | Percentage of projects completed within the programme. |
| Projects Linked to Programme | Number of projects properly linked to the programme. |
| Projects Without Programme Link | Number of projects requiring programme alignment. |
| Programme Budget Utilization % | Tracks budget consumption across programme projects. |
| Programme Schedule Variance | Shows whether the programme is ahead or behind schedule. |
| Programme Benefits Progress | Tracks delivery of expected programme outcomes and benefits. |
| Dependency Health | Measures whether inter-project dependencies are healthy, delayed, or blocked. |
| Change Request Count | Shows approved, pending, and rejected changes. |
| Resource Capacity Pressure | Indicates whether teams are overloaded or under-resourced. |
| Issue Escalation Count | Number of issues escalated from projects to programme level. |
| Milestone Achievement Rate | Percentage of major programme milestones achieved. |
| Programme Governance Compliance % | Shows whether programme-level documentation and approvals are complete. |

---

## 2.2 Programme Alert Metrics

| Alert | Why It Matters |
|---|---|
| Programme Behind Schedule | Warns that multiple projects may be affecting programme outcomes. |
| Programme Budget Pressure | Shows financial exposure across grouped projects. |
| Blocked Dependencies | Highlights projects waiting on other projects, vendors, or decisions. |
| Project Without Programme Link | Shows governance or hierarchy gaps. |
| Critical Project Within Programme | Flags one failing project that may damage the whole programme. |
| Benefits at Risk | Programme may deliver outputs but fail to achieve intended outcomes. |
| Resource Conflict | Same people are assigned to too many projects. |
| Too Many Scope Changes | Indicates instability or weak upfront definition. |
| Escalated Issues Overdue | Programme-level problems are not being resolved quickly. |
| Milestones Missed | Key programme milestones have slipped or failed. |
| Programme Governance Gap | Missing approvals, reports, or control documentation. |

# 3. Project Dashboard Metrics

Project-level metrics should help executives answer:

> Is this project under control, and does it need intervention?

---

## 3.1 Project Summary Metrics

| Metric | Purpose |
|---|---|
| Total Projects | Shows total number of projects in the platform. |
| Active Projects | Shows projects currently in execution. |
| Planned / Draft Projects | Shows projects still being prepared. |
| Completed Projects | Shows projects formally closed. |
| Project Health Score | Overall score based on scope, schedule, cost, risk, quality, and governance. |
| Schedule Status | Shows whether the project is on track, delayed, ahead, or has no baseline. |
| Budget Status | Shows whether the project is on budget, overspending, underspending, or has no budget. |
| Scope Stability | Measures approved changes against original scope. |
| Milestone Progress % | Tracks completion of key project milestones. |
| Task Completion % | Shows actual delivery progress. |
| Overdue Tasks | Highlights immediate delivery pressure. |
| Critical Path Delay | Shows whether key tasks are threatening the end date. |
| Open Risks | Total unresolved risks, especially high and critical risks. |
| Open Issues | Total unresolved project issues. |
| Overdue Issues | Issues not closed within agreed SLA. |
| Change Requests Pending | Pending changes awaiting approval. |
| Document Compliance % | Required project documents completed and approved. |
| Approval Status | Shows whether key governance approvals are pending. |
| Team Allocation % | Shows whether the team is under-allocated, fully allocated, or overloaded. |
| Vendor / External Dependency Status | Flags delays caused by vendors or third parties. |
| Last Status Update Date | Shows whether the project is being actively reported. |
| Project Linkage Status | Shows whether project is linked to a programme, portfolio, both, or neither. |

---

## 3.2 Project Alert Metrics

| Alert | Why It Matters |
|---|---|
| Project Has No Baseline | No reliable schedule or budget control exists. |
| Project Behind Schedule | Delivery date may be at risk. |
| Project Over Budget | Indicates financial control issue. |
| Critical Risk Open | Requires management attention. |
| Blocked Tasks | Work cannot continue without intervention. |
| Overdue Milestone | Key project commitment has been missed. |
| Pending Approval | Project may be stuck waiting for governance decision. |
| Too Many Scope Changes | Indicates scope creep or unstable requirements. |
| No Recent Update | Project reporting discipline issue. |
| Unassigned Tasks | Work exists but has no clear owner. |
| Project Not Linked to Programme / Portfolio | Governance hierarchy gap. |
| Low Team Capacity | Delivery team may not be sufficient. |
| Vendor Delay | External party is delaying progress. |
| Quality Failure | Quality checks, testing, or acceptance criteria are failing. |
| High Issue Age | Issues remain open for too long. |

# 4. Recommended Executive Summary Layout

For quick decision-making, each module should have four layers.

---

## 4.1 Layer 1: Count Summary

This is the basic operational view already visible on the dashboard.

| Area | Example Metrics |
|---|---|
| Portfolio | Total portfolios, active, completed, on hold, planning, cancelled. |
| Programmes | Total programmes, active, planning, linked to portfolio, unlinked. |
| Projects | Total projects, active, planned, completed, linked to programme, unlinked. |

---

## 4.2 Layer 2: Health Summary

This layer gives executives a fast performance view.

| Area | Recommended Health Metrics |
|---|---|
| Portfolio | Portfolio Health Index, Strategic Alignment, Benefits Realization. |
| Programme | Programme Health Index, Delivery Progress, Dependency Health. |
| Project | Project Health Score, Schedule Status, Budget Status, Scope Stability. |

---

## 4.3 Layer 3: Risk and Alert Summary

This is the most important executive decision layer.

| Alert Type | Example Metric |
|---|---|
| Critical Risks | Number of high or critical risks open. |
| Overdue Decisions | Number of decisions pending beyond SLA. |
| Blocked Dependencies | Number of dependencies stopping delivery. |
| Schedule Exceptions | Number of projects or programmes behind schedule. |
| Budget Exceptions | Number of items over budget or with funding gaps. |
| Governance Gaps | Missing approvals, reports, or documents. |
| No Recent Update | Items not updated within the required reporting cycle. |
| Unlinked Items | Programmes or projects not properly linked in the hierarchy. |

---

## 4.4 Layer 4: Action Summary

This layer tells leadership where to focus.

| Action Metric | Purpose |
|---|---|
| Needs Executive Decision | Items requiring senior approval or direction. |
| Needs PM Attention | Items the project manager must resolve. |
| Needs Programme Intervention | Cross-project issues requiring programme-level action. |
| Needs Portfolio Realignment | Strategic, funding, or prioritization correction required. |
| Needs Governance Review | Missing documents, approvals, or control actions. |
| Needs Escalation | Items that have exceeded agreed tolerance or SLA. |

# 8. Recommended RAG Logic

Use simple Red, Amber, Green logic to make the dashboard easier for executives.

---

## 8.1 Portfolio RAG Logic

| Status | Suggested Rule |
|---|---|
| Green | Health index above 80%, benefits tracking, no critical governance gaps. |
| Amber | Health index between 60% and 80%, some delayed decisions or minor budget pressure. |
| Red | Health index below 60%, benefits at risk, budget exception, or major unresolved risks. |

---

## 8.2 Programme RAG Logic

| Status | Suggested Rule |
|---|---|
| Green | Most projects on track, dependencies healthy, milestones being achieved. |
| Amber | Some projects delayed, dependencies under pressure, minor budget or resource concerns. |
| Red | Multiple projects critical, blocked dependencies, missed milestones, or budget breach. |

---

## 8.3 Project RAG Logic

| Status | Suggested Rule |
|---|---|
| Green | On schedule, on budget, risks controlled, documents approved. |
| Amber | Minor schedule slippage, manageable issues, some approvals pending. |
| Red | Major delay, budget breach, critical risks open, blocked work, or missing baseline. |

---

# 9. Final Recommendation

The dashboard should move from:

> How many portfolios, programmes, and projects do we have?

To:

> Where should leadership focus today?

The strongest executive metrics are:

- Health
- Budget
- Schedule
- Benefits
- Risks
- Dependencies
- Decisions
- Governance
- Linkage
- Recent updates

Together, these metrics turn the platform dashboard into an **executive command centre** for fast, informed decision-making.
