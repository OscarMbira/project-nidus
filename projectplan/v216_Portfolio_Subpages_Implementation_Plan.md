## Architecture

Each new page follows the same pattern:
- DB: direct `platformDb` queries joining `portfolio_*` tables with `portfolios`
- UI: dark-first, theme-aware, responsive
- Features: search, filter by portfolio, full CRUD with modals, success confirmation, export (Excel/Word/PPT/CSV/XML/JSON/Print)
- All export uses the existing `ExportListMenu` / `ExportRecordMenu` utilities

## Page-by-Page Spec

### 1. Portfolio Dashboard (`/platform/portfolio/dashboard`)
**Purpose:** Executive overview across ALL portfolios

KPI Cards (top row):
- Total Portfolios | Active Portfolios | Total Budget (sum) | Avg Health Score

Portfolio Health Table:
- Name | Code | Status | Projects Count | Budget | Health Score | Owner

### 2. Portfolio Projects (`/platform/portfolio/projects`)
**Purpose:** All projects across all portfolios

Query: `portfolio_projects` → JOIN `projects` → JOIN `portfolios`

Columns: Project Name | Project Code | Portfolio | Status | Methodology | Start | End

CRUD:
- Add: pick an existing project + pick a portfolio (searchable dropdowns)
- Remove: soft-delete the portfolio_projects row (set assignment_status = 'removed')

Filters: portfolio dropdown, status dropdown, search text

### 3. Portfolio Resources (`/platform/portfolio/resources`)
**Purpose:** All team members/resources across all portfolios

Query: `portfolio_members` → JOIN `users` → JOIN `portfolios`

Columns: Member Name | Email | Role | Portfolio | Status | Start Date | End Date

CRUD:
- Add Member: pick portfolio + pick user + set role + optional dates
- Edit: update role / dates
- Remove: set assignment_status = 'inactive'

Filters: portfolio dropdown, role dropdown, status dropdown, search

### 4. Portfolio Financial (`/platform/portfolio/financial`)
**Purpose:** Budget & financial overview across all portfolios

Query: `portfolio_budgets` → JOIN `portfolios`

Summary cards:
- Total Approved Budget | Total Spent | Total Remaining | Avg Utilisation %

Table Columns: Portfolio | Budget Name | Type | Approved | Spent | Remaining | Utilisation % | Status

CRUD:
- Add Budget: portfolio + name + type + amounts + dates + currency + approval
- Edit inline / modal
- Soft delete

Filters: portfolio, budget type, status, currency

### 5. Portfolio Reports (`/platform/portfolio/reports`)
**Purpose:** All reports across all portfolios

Query: `portfolio_reports` → JOIN `portfolios`

Columns: Report Name | Type | Portfolio | Period | Status | Generated At | Generated By

CRUD:
- Create Report: portfolio + name + type + period + content fields
- View/Edit: open modal with full fields
- Delete: soft delete

Filters: portfolio, report type, status, date range

### 6. Portfolio Governance (`/platform/portfolio/governance`)
**Purpose:** Governance settings for each portfolio

Query: `portfolio_governance` → JOIN `portfolios`

Columns: Portfolio | Governance Model | Review Frequency | Last Review | Next Review | Decision Authority

CRUD:
- Edit: open governance record for that portfolio in a modal
- Creates blank governance record if none exists for a portfolio

Filters: portfolio, governance model
